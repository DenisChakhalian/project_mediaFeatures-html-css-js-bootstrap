Синтаксис

Синтаксически медиа-запросы напоминают свойства CSS: они имеют имена и принимают определенные значения. Однако есть несколько важных различий между свойствами и функциями мультимедиа:

  • Свойства используются в объявлениях для предоставления информации о том, как представить документ. Медиа-запросы используются в выражениях для описания требований устройства вывода.
  • Большинство медиа-запросов принимают необязательные префиксы «min» или «max» для выражения ограничений «больше или равно» и «меньше или равно». Данный синтаксис используется для исключения символов «<» и «>», которые могут быть не совместимы с HTML и XML. Те медиа-запросы, которые принимают префиксы, чаще всего будут использоваться с префиксами, но также могут использоваться отдельно.
  • Свойства всегда требуют значения для формирования объявления. Медиа-запросы, с другой стороны, также могут использоваться без значения. Для медиа-запросов функция (feature) будет иметь значение true, если (feature: x) будет иметь значение true для значения x, отличного от нуля или ноль, за которым следует идентификатор устройства (т. е., кроме 0, 0px, 0em и т. д.). Медиа-запросы с префиксом min / max нельзя использовать без значения. Когда медиа-запрос с префиксом min / max используется без значения, это делает медиа-запрос искаженным.
  • Свойства могут принимать более сложные значения, например, вычисления, которые включают несколько других значений. Медиа-запросы допускают только одно значение: одно ключевое слово, один номер или номер с идентификатором устройства. (Единственными исключениями являются мультимедийные функции «aspect-ratio» и «device-aspect-ratio».)

Все запросы начинаются с правила @media, после чего следует условие, в котором используются типы носителей, логические операторы и медиа-функции.

Типы носителей
Тип Описание
all Все типы. Это значение используется по умолчанию.
braille Устройства, основанные на системе Брайля, которые предназначены для чтения слепыми людьми.
embossed Принтеры, использующие для печати систему Брайля.
handheld Смартфоны и аналогичные им аппараты.
print Принтеры и другие печатающие устройства.
projection Проекторы.
screen Экран монитора.
speech Речевые синтезаторы, а также программы для воспроизведения текста вслух. Сюда, например, можно отнести речевые браузеры.
tty Устройства с фиксированным размером символов (телетайпы, терминалы, устройства с ограничениями дисплея).
tv Телевизоры.

Медиа запросы позволяют адаптировать страницу для различных типов устройств, таких как: принтеры, речевых браузеров, устройств Брайля, телевизоров и так далее. Например это правило для принтеров:

@media print { ... }

Вы также можете писать правила сразу для нескольких устройств. Например этот @media написан сразу для экранов и принтеров:

@media screen, print { ... }

Список устройств вы можете найти в таблице выше. Но для задания более детальных и узконаправленных правил вам нужно просмотреть раздел "Примеры".
