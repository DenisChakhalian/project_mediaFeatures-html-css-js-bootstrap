Переменные

width

Значение: <длина>.

Относится к: визуальным и тактильным типам носителей.

Принимает min/max префиксы: да.

Медиа-функция «width» описывает ширину целевой области отображения устройства вывода. Для непрерывного носителя это ширина области просмотра, включая размер визуализированной полосы прокрутки (если есть). Для постраничных носителей это ширина поля страницы. Указанная <длина> не может быть отрицательной.

Пример 1
Например, этот медиа-запрос выражает, что таблица стилей пригодна для печати на выходе шире, чем 25см:
<link rel="stylesheet" media="print and (min-width: 25cm)" href="http://…" />
Пример 2
Этот медиа-запрос выражает, что таблица стилей может использоваться на устройствах с областью просмотра (часть экрана/бумаги, на которой отображается документ) шириной от 400 до 700 пикселей:
@media screen and (min-width: 400px) and (max-width: 700px) { … }
Пример 3
Этот медиа-запрос выражает, что таблица стилей может использоваться на экране и портативных устройствах, если ширина области просмотра превышает 20em.
@media handheld and (min-width: 20em), screen and (min-width: 20em) { … }

height

Значение: <длина>.

Относится к: визуальным и тактильным типам носителей.

Принимает min/max префиксы: да.

Медиа-функция «height» описывает высоту целевой области отображения устройства вывода. Для непрерывных носителей это высота окна просмотра, включая размер отображаемой полосы прокрутки (если есть). Для выгружаемых носителей это высота окна страницы. Указанная <длина> не может быть отрицательной.

Пример 1
Например, этот медиа-запрос выражает, что таблица стилей пригодна для печати на выходе шире, чем 50 пикселей:
<link rel="stylesheet" media="print and (min-height: 50px)" href="http://…" />
Пример 2
Этот медиа-запрос выражает, что таблица стилей может использоваться на устройствах с областью просмотра (часть экрана/бумаги, на которой отображается документ) шириной от 200 до 450 пикселей:
@media screen and (min-height: 200px) and (max-height: 450px) { … }

device-width

Значение: <длина>.

Относится к: визуальным и тактильным типам носителей.

Принимает min/max префиксы: да.

Медиа-функция «device-width» описывает ширину поверхности рендеринга устройства вывода. Для непрерывных носителей это ширина экрана. Для страничных носителей это ширина размера листа страницы. Указанная <длина> не может быть отрицательной.

Пример 1
Таблица стилей будет применяться только к экранам, на которых в настоящий момент отображается ровно 800 пикселей по горизонтали.
@media screen and (device-width: 800px) { … }

device-height

Значение: <длина>.

Относится к: визуальным и тактильным типам носителей.

Принимает min/max префиксы: да.

Медиа-функция «device-height» описывает высоту рендеринга поверхности устройства вывода. Для непрерывного носителя это высота экрана. Для выгружаемых носителей это высота размера листа страницы. Указанная <длина> не может быть отрицательной.

Пример 1
Таблица стилей будет применяться только к экранам, которые имеют ровно 600 пикселей по вертикали.
<link rel="stylesheet" media="screen and (device-height: 600px)" />

orientation

Значение: portrait | landscape.

Относится к: типам растровых носителей.

Принимает min/max префиксы: нет.

Медиа-функция «orientation» принимает значение portrait, если значение медиа-функции height больше или равно значению медиа-функции width. В противном случае orientation принимает значение landscape.

Пример 1
@media all and (orientation:portrait) { … }
Пример 2
@media all and (orientation:landscape) { … }

aspect-ratio

Значение: <соотношение>.

Относится к: типам растровых носителей.

Принимает min/max префиксы: да.

Медиа-функция «aspect-ratio» определяется как отношение значения медиа-функции «width» к значению медиа-функции «height».

Пример 1
@media (min-aspect-ratio: 8/5) { … }

device-aspect-ratio

Значение: <соотношение>.

Относится к: типам растровых носителей.

Принимает min/max префиксы: да.

Медиа-функция «device-aspect-ratio» определяется как отношение значения медиа-функции «device-width» к значению медиа-функции «device-height».

Пример 1
Например, если экранное устройство с квадратными пикселями имеет 1280 пикселей по горизонтали и 720 пикселей по вертикали (обычно называемые «16: 9»), следующие медиа-запросы будут соответствовать такому устройству:
@media screen and (device-aspect-ratio: 16/9) { … }
@media screen and (device-aspect-ratio: 32/18) { … }
@media screen and (device-aspect-ratio: 1280/720) { … }
@media screen and (device-aspect-ratio: 2560/1440) { … }

color

Значение: <целое число>.

Относится к: типам визуальных носителей.

Принимает min/max префиксы: да.

Медиа-функция «color» описывает количество бит на компонент цвета устройства вывода. Если устройство не является цветным устройством, значение равно нулю.

Пример 1
Например, эти два медиа-запроса выражают, что таблица стилей применяется ко всем цветным устройствам:
@media all and (color) { … }
@media all and (min-color: 1) { … }
Пример 2
Этот медиа-запрос выражает, что таблица стилей применяется к цветным устройствам с 2 или более битами на компонент цвета:
@media all and (min-color: 2) { … }

color-index

Значение: <целое число>.

Относится к: типам визуальных носителей.

Принимает min/max префиксы: да.

Медиа-функция «color-index» описывает количество записей в таблице соответствия цветов устройства вывода. Если устройство не использует таблицу соответствия цветов, значение равно нулю. Указанное <целое число> не может быть отрицательным.

Пример 1
Например, вот два способа выразить, что таблица стилей применяется ко всем устройствам с индексами цвета:
@media all and (color-index) { … }
@media all and (min-color-index: 1) { … }
Пример 2
Этот медиа-запрос выражает, что таблица стилей применяется к устройству индекса цвета с 256 или более записями:
<?xml-stylesheet media="all and (min-color-index: 256)" href="http://www.example.com/…"?>

monochrome

Значение: <целое число>.

Относится к: типам визуальных носителей.

Принимает min/max префиксы: да.

Медиа-функция «monochrome» описывает количество бит на пиксель в монохромном буфере кадров. Если устройство не является монохромным устройством, значение устройства вывода будет 0. Указанное <целое число> не может быть отрицательным.

Пример 1
Например, вот два способа выразить, что таблица стилей применяется ко всем монохромным устройствам:
@media all and (monochrome) { … }
@media all and (min-monochrome: 1) { … }
Пример 2
Таблица стилей применяется к монохромным устройствам с более чем 2 битами на пиксель:
@media all and (min-monochrome: 2) { … }
Пример 3
Таблица стилей для цветных страниц и другая для монохромных:
<link rel="stylesheet" media="print and (color)" href="http://…" />
<link rel="stylesheet" media="print and (monochrome)" href="http://…" />

resolution

Значение: <разрешение>.

Относится к: типам растровых носителей.

Принимает min/max префиксы: да.

Медиа-функция «resolution» описывает разрешение устройства вывода, то есть плотность пикселей. При запросе устройств с неквадратными пикселями в запросах «минимальное разрешение» наименее плотное измерение должно сравниваться с указанным значением, а в запросах «max-resolution» вместо этого следует сравнивать наиболее плотное измерение. Запрос «resolution» (без префикса «min-» или «max-») никогда не сопоставляет устройство с неквадратными пикселями.

Пример 1
Например, этот медиа-запрос выражает, что таблица стилей может использоваться на устройствах с разрешением более 300 точек на дюйм:
@media print and (min-resolution: 300dpi) { … }
Пример 2
Этот медиа-запрос выражает, что таблица стилей может использоваться на устройствах с разрешением более 118 точек на сантиметр:
@media print and (min-resolution: 118dpcm) { … }

scan

Значение: progressive | interlace.

Относится к: типам "tv" медиа.

Принимает min/max префиксы: нет.

Медиа-функция «scan» описывает процесс сканирования выходных телевизионных устройств.

Пример 1
Например, этот медиа-запрос выражает, что таблица стилей может использоваться на телевизионных устройствах с прогрессивной разверткой:
@media tv and (scan: progressive) { … }

grid

Значение: <целое число>.

Относится к: типам визуальных и тактильных носители.

Принимает min/max префиксы: нет.

Медиа-функция «grid» используется для запроса, является ли устройство вывода сеткой или растровым изображением. Если устройство вывода основано на сетке (например, терминал "tty" или дисплей телефона только с одним фиксированным шрифтом), значение будет равно 1. В противном случае значение будет равно 0.

Пример 1
@media handheld and (grid) and (max-width: 15em) { … }
Пример 2
@media handheld and (grid) and (device-max-height: 7em) { … }
